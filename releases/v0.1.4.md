### Proper Job Control

Foreground jobs now get their own process group and terminal control via `tcsetpgrp`. This means Ctrl+C and Ctrl+Z are delivered only to the running command, not the shell. `fg` correctly resumes stopped jobs with terminal ownership.

Previously, foreground jobs shared the shell's process group, which could cause signals to hit the shell or get swallowed.

### Faster Prompt

Git status widgets (`branch`, `staged`, `modified`, `ahead/behind`, `diff stats`) are now collected on a background thread. The prompt renders instantly from a cached snapshot instead of blocking on `git status` and `git diff` subprocesses on every keypress.

If you noticed a delay on Ctrl+C or pressing Enter on an empty line — that's gone.
