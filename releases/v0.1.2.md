### Bundled Lua 5.5

Lua is now compiled from source and statically linked into the binary. No more system Lua dependency — `brew install xyron` gives you a single self-contained binary that just works.

This also upgrades from Lua 5.4 to Lua 5.5. Your existing `config.lua` should work without changes.

### Project System

Drop a `xyron.toml` in your repo root to define project commands, environment, services, and secrets.

```toml
[project]
name = "api"

[commands]
dev = "bun run dev"
test = "bun test"

[env]
sources = [".env", ".env.local"]

[services.web]
command = "bun run dev"
```

- **Auto-activation** — `cd` into a project directory and the environment loads automatically. `cd` out and it's removed cleanly.
- **Commands** — `xyron run dev` executes with the resolved environment.
- **Services** — `xyron up` / `down` / `ps` / `logs -f` manage background dev services as detached daemons that survive shell sessions.
- **Doctor** — `xyron doctor` validates env files, required secrets, command paths, service state, and git status in one scan.
- **Context engine** — `xyron context explain DATABASE_URL` shows the full provenance chain of any env value.
- **Bootstrap** — `xyron init` detects your ecosystem and generates `xyron.toml`. `xyron new bun my-app` scaffolds via native tooling.

### Secrets Manager

GPG-encrypted secret storage with `xyron secrets add`, `get`, `list`, and `open` (TUI browser).

Use `${secret:NAME}` in `xyron.toml` env values — secrets are resolved at runtime and auto-redacted in explain output.
