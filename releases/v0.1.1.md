Xyron is a modern shell that replaces bash, zsh, and fish. Built from scratch in Zig with Lua scripting, SQLite history, structured output, and a built-in project runtime. Not POSIX — by design. Under 2MB.

Works standalone in any terminal. Works best inside [Attyx](https://github.com/semos-labs/attyx), where it becomes a runtime backend with native UI integration.

This is the first public release.

---

### Shell

**Lua, not bash.** Configuration, hooks, custom commands, and automation — all in `~/.config/xyron/config.lua`. Full LSP support with auto-installed type definitions. A migration assistant converts your existing bash/zsh config.

**SQLite history.** Every command recorded with exit code, duration, working directory, and timestamp. Query it like a database: `history failed`, `history slow 5000`, `history cwd ~/Projects`. Ctrl+R opens a full-screen fuzzy search. Ghost text suggests from history as you type.

**Structured output.** `ls`, `ps`, `env`, `jobs`, `history` return colored tables with auto-sized columns — not raw text. Chain them: `ps | where %cpu > 5 | sort %cpu desc | select pid,command`. Parse external data with `json`, `csv`, `to_json`. No awk/sed/jq needed.

**Smart completions.** Fuzzy picker with descriptions. Parses `--help` output and caches flags. Path, env, and command providers.

**Vim mode.** Normal, insert, and visual modes. Text objects (`ciw`, `da"`, `yi(`), visual selection with highlighting, system clipboard via OSC 52. Cursor shape changes with mode.

**Prompt engine.** Composable segments — cwd, git branch + status, job count, command duration, project name — with Lua-defined custom segments, multiline layout, and right-alignment via spacers.

**Directory jumping.** Built-in `j` command with frecency ranking. Learns from `cd`. Import from zoxide.

**Job control.** Background `&`, Ctrl+Z suspend, `fg`/`bg` resume, process groups.

**Fuzzy finder.** Built-in `fz` for file and directory search. Zero dependencies.

**Secrets manager.** GPG-encrypted store. `xyron secrets add`, `get`, `list`, `open` (TUI browser).

### Project System

Drop a `xyron.toml` in your repo and Xyron handles environment, commands, services, and diagnostics.

```toml
[project]
name = "api"

[commands]
dev = "bun run dev"
test = "bun test"

[env]
sources = [".env", ".env.local"]

[env.values]
DATABASE_URL = "postgres://user:${secret:DB_PASS}@localhost/mydb"

[secrets]
required = ["API_KEY"]

[services.web]
command = "bun run dev"
```

**Smart entry.** `cd` into a project — environment activates automatically with a one-line status. `cd` out — overlay removed cleanly. Prompt shows active project and missing secret count.

**Context engine.** Deterministic merge: system env → `.env` files → manifest `[env.values]` → overrides. Every value tracks where it came from. `xyron context explain DATABASE_URL` shows the full provenance chain.

**Secret interpolation.** `${secret:NAME}` in `[env.values]` resolves from the encrypted secrets store at runtime. Auto-redacted in explain output.

**Commands.** `xyron run dev` executes with the resolved environment. Shows available commands when the name doesn't match.

**Services.** `xyron up` / `down` / `ps` / `logs -f` — manage background dev services as detached daemons. Survive shell sessions. Detect context staleness.

**Doctor.** `xyron doctor` validates env files, required secrets, command paths, service state, and git status in one scan.

**Bootstrap.** `xyron init` detects your ecosystem (Bun, Node, Rust, Go, Zig, Python) and generates `xyron.toml`. `xyron new bun my-app` scaffolds via native tooling then adds Xyron setup.

### Attyx Integration

When running inside the [Attyx terminal](https://github.com/semos-labs/attyx), Xyron provides:
- IPC socket for external queries (completions, history, state)
- Headless mode with binary protocol for frontend rendering
- OSC 7339 events broadcasting command lifecycle
- Native popups, file pickers, and structured overlays

### Install

```
brew tap semos-labs/tap
brew install xyron
```

Set as default shell:
```
echo "$(which xyron)" | sudo tee -a /etc/shells
chsh -s $(which xyron)
```
