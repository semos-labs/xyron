Xyron is a development shell built in Zig. It replaces bash/zsh/fish with a project-aware runtime — your shell understands what you're working on, manages your environment, runs your services, and tells you what's broken.

This is the first public release.

### What Is This

A standalone shell where projects are first-class. Drop a `xyron.toml` in your repo and Xyron handles environment, commands, services, and diagnostics. No plugins, no dotfile frameworks, no manual wiring.

- **Not POSIX** — Lua for scripting, not bash
- **Not a wrapper** — standalone binary, replaces your shell entirely
- **Not a container tool** — lightweight process manager for dev services
- **Pairs with [Attyx](https://github.com/semos-labs/attyx)** — deep integration when running inside the Attyx terminal, works standalone everywhere else

### Project System

```toml
[project]
name = "api"

[commands]
dev = "bun run dev"
test = "bun test"

[env]
sources = [".env", ".env.local"]

[env.values]
DATABASE_URL = "postgres://user:${secret:DB_PASS}@localhost/mydb"

[secrets]
required = ["API_KEY"]

[services.web]
command = "bun run dev"
```

- `cd` into a project — context activates automatically, environment applied
- `cd` out — overlay removed, shell returns to baseline
- `xyron init` — detects ecosystem (Bun, Node, Rust, Go, Zig, Python) and generates config
- `xyron new bun my-app` — scaffolds via native tooling, adds xyron.toml

### Commands & Services

- `xyron run dev` — execute project commands with resolved environment
- `xyron up` / `down` / `ps` / `logs` — manage background services as detached daemons
- Services survive shell sessions, track context staleness, scoped per-project

### Diagnostics

- `xyron doctor` — validates env files, secrets, command paths, service state, git status in one scan
- `xyron context explain DATABASE_URL` — shows exactly where a value came from, which source won, full override chain
- Sensitive values auto-redacted (keys containing SECRET, TOKEN, PASSWORD, etc.)

### Environment Engine

Deterministic merge with full provenance:
1. System env (lowest)
2. `.env` files in declaration order
3. `[env.values]` with `${secret:NAME}` interpolation from GPG-encrypted store
4. Overrides (highest)

Every value tracks its source. No guessing.

### Shell

- **Lua config** with LSP support — type definitions auto-installed, full autocomplete in your editor
- **SQLite history** — every command recorded with cwd, exit code, duration. Queryable.
- **Structured output** — `ls`, `ps`, `env`, `json`, `csv` return typed tables with auto-sized columns
- **Prompt segments** — configurable via Lua: cwd, git status, project name, duration, jobs, vim mode
- **Vim mode** — normal/insert/visual, text objects (`ciw`, `da"`, `yi(`), system clipboard
- **Ctrl+R** — full-screen fuzzy history search
- **Ghost text** — inline suggestions from history, ordered by recency
- **Job control** — background/foreground, Ctrl+Z suspend, fg/bg resume
- **Migration assistant** — converts bash/zsh config to Lua

### Install

```
brew tap semos-labs/tap
brew install xyron
```

Then add to your shell list and switch:
```
echo "$(which xyron)" | sudo tee -a /etc/shells
chsh -s $(which xyron)
```
