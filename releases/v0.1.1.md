## Xyron 0.1.1 — Project-Aware Shell

First public release of Xyron, a standalone shell built in Zig with Lua scripting and a built-in project runtime.

### Project System

Xyron turns directories into environments. Drop a `xyron.toml` in your project root and Xyron handles the rest.

- **`xyron init`** — detect your ecosystem (Bun, Node, Rust, Go, Zig, Python) and generate a baseline `xyron.toml`
- **`xyron new <eco> <name>`** — scaffold a new project with native tooling, then apply Xyron setup
- **Smart entry** — `cd` into a project and context activates automatically, `cd` out and it cleans up
- **Prompt widget** — `xyron_project` segment shows active project name and missing secrets count

### Commands & Services

- **`xyron run <cmd>`** — run project-defined commands with the correct environment
- **`xyron up / down / restart / ps / logs`** — manage long-running dev services as detached daemons
- Services are project-scoped, survive shell sessions, and track context staleness

### Diagnostics

- **`xyron doctor`** — validate env files, required secrets, command cwd, service state, git status
- **`xyron context explain [KEY]`** — trace where any env value came from, full provenance chain, automatic redaction for sensitive keys

### Environment & Secrets

- **Context engine** — deterministic merge of system env → `.env` files → `[env.values]` → overrides
- **`[env.values]`** with `${secret:NAME}` interpolation — reference secrets from `xyron secrets` directly in `xyron.toml`
- **Provenance tracking** — every value knows its source, whether it was overridden, and by what

### Editor

- **Vim mode** — normal, insert, and visual modes with `xyron.config.vim_mode(true)`
- **Text objects** — `ciw`, `diw`, `da"`, `yi(`, `va[`, and more
- **Visual selection** — `v` + motions/text objects, then `d`/`c`/`y` to operate
- **System clipboard** — yank copies to clipboard via OSC 52 + pbcopy

### Shell Features

- Lua scripting via `config.lua` with full `xyron.*` API
- Configurable prompt with built-in segments (cwd, git, jobs, duration, vim mode)
- Tab completion, fuzzy history search (Ctrl+R), ghost text suggestions
- Structured output (`ls`, `ps`, `env`, `json`, `csv`, `query`)
- Job control, aliases, command correction
- Attyx terminal integration (IPC, native UI bridge)

### Lua API

- Type definitions auto-installed for LuaLS — full autocomplete and docs in your editor
- Config toggles under `xyron.config.*` (completion, vim_mode, block_ui)
- `xyron.project_info()` — access active project state from Lua
