### Instant Loading

Xyron now shows the prompt immediately on startup. Slow init work — like sourcing `nvm.sh` or other bash scripts — runs in the background via `xyron.defer()` after the first prompt is already visible.

Two new Lua APIs support this:

- **`xyron.defer(fn)`** — queue a function to run after the first prompt renders
- **`xyron.lazy_command(name, module)`** — register a command that defers `require(module)` until first use

There's also `xyron.source(script, opts)` for sourcing bash scripts and importing their environment variables into xyron, and `xyron.capture(cmd)` for running a command and capturing its output (replaces `io.popen`, which is now disabled to prevent fork deadlocks with background threads).

### Powerline Prompt Style

The prompt system now supports powerline-style rendering with colored segments and custom separators (, , or any glyph). Configure it from Lua with structured `fg`/`bg` colors per segment instead of raw ANSI strings.

Use `xyron.prompt.init({ separator = "" })` to enable powerline mode, then add segments with `fg` and `bg` color fields.

### Terminal Title

The terminal tab/window title now updates automatically — showing the running command while it executes, and `xyron: ~/path` when idle. Works in any terminal that supports OSC 0.

### Proper Job Control

Every job (foreground and background) now gets its own process group. Foreground jobs receive terminal control via `tcsetpgrp`, so Ctrl+C and Ctrl+Z target only the running command. The shell reclaims the terminal after the job finishes or stops.
